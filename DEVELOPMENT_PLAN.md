## 一、项目概述

### 1.1 当前项目状态

**TradingAgents** 是一个基于 LLM 的多 Agent 金融交易框架，当前版本 v0.2.3，具有以下核心特性：

- **Agent 团队架构**:
  - 基本面分析师 (Fundamentals Analyst)
  - 情绪分析师 (Sentiment Analyst)
  - 新闻分析师 (News Analyst)
  - 技术分析师 (Technical Analyst)
  - 多头/空头研究员 (Bull/Bear Researcher)
  - 交易员 (Trader)
  - 风险管理团队 (Risk Management)
  - 投资组合经理 (Portfolio Manager)

- **支持的数据源**:
  - yfinance (默认)
  - Alpha Vantage

- **支持的 LLM 提供商**:
  - OpenAI (GPT-5.4)
  - Google (Gemini 3.x)
  - Anthropic (Claude 4.x)
  - xAI (Grok 4.x)
  - OpenRouter
  - Ollama (本地模型)

- **当前支持的市场**: 主要针对美股设计

### 1.2 目标

扩展 TradingAgents 以支持以下市场：
- **美股** (US Stock Market) - 扩展现有功能
- **比特币** (Bitcoin/Crypto) - 新增加密货币支持
- **港股** (HK Stock Market) - 新增港股支持
- **A股** (China A-Share) - 新增A股支持

### 1.3 核心要求

> ⚠️ **强制要求 - 项目必须满足以下条件**

#### 1.3.1 数据真实性要求

- ✅ **所有数据必须基于真实数据源**
  - 禁止使用仿真或模拟数据进行功能演示

  - 回测系统必须使用真实历史数据进行验证
  
- ✅ **分析结果必须基于真实市场数据**
  - Agent 的投资分析必须基于实时/历史真实行情
  - 技术指标、基本面数据、新闻情绪必须来自真实数据源

#### 1.3.2 Shell 便捷运行界面要求

- ✅ **提供交互式 CLI 入口**
  - 支持 `python main.py` 或专用脚本启动交互式菜单
  - 支持功能选择: 市场切换、数据查询、策略回测等
  - 支持快捷命令: 直接指定股票代码和市场进行查询

- ✅ **功能模块**

#### 1.3.3 测试框架要求

- ✅ **一键测试脚本**
  - 提供 `python run_tests.py` 或 `bash run_tests.sh` 一键运行所有测试

## 九、编程规范

### 9.1 代码风格

- **语言版本**: Python 3.13+
- **代码规范**: 遵循 [PEP 8](https://www.python.org/dev/peps/pep-0008/) 风格指南
- **类型注解**: 使用类型注解 (Type Hints) 增强代码可读性和可维护性
- **Docstring**: 所有公共类和函数必须包含 docstring，使用 Google 或 NumPy 风格
- **命名规范**:
  - 类名: `PascalCase` (如 `TradingAgentsGraph`)
  - 函数/方法: `snake_case` (如 `get_stock_data`)
  - 常量: `UPPER_SNAKE_CASE` (如 `MAX_DEBATE_ROUNDS`)
  - 私有属性: `_leading_underscore`

### 9.2 代码质量要求

- **类型检查**: 使用 mypy 进行类型检查，类型注解覆盖率达到 90% 以上
- **代码格式化**: 使用 Black 进行代码格式化，行长度限制 100
- **Import 排序**: 使用 isort 进行 import 排序
- **代码审查**: 提交前必须通过 pylint 检查 (评分 9.0/10 以上)
- **测试覆盖**: 核心业务逻辑测试覆盖率达到 80% 以上

### 9.3 日志规范

```python
import logging

# 创建模块级 logger
logger = logging.getLogger(__name__)

# 日志级别使用:
# - DEBUG: 详细调试信息
# - INFO: 正常业务流程
# - WARNING: 警告但不影响功能
# - ERROR: 错误导致部分功能失败
# - CRITICAL: 严重错误导致程序崩溃
```

### 9.4 异常处理

- 使用自定义异常类，便于错误定位和处理
- 捕获具体异常类型，避免捕获 `Exception` 或 `BaseException`
- 异常信息包含足够的上下文信息
- 日志记录异常堆栈信息

## 十一、GitHub 代码维护

### 11.1 分支策略

```
main (保护分支)
    ↑
    └─ develop (开发分支)
            ↑
            ├─ feature/xxx (功能分支)
            ├─ bugfix/xxx (修复分支)
            └─ hotfix/xxx (紧急修复分支)
```

- **main 分支**: 生产环境代码，受到保护，禁止直接推送
- **develop 分支**: 开发主分支，整合所有功能分支
- **feature 分支**: 功能开发分支，命名: `feature/功能名-开发者`
- **bugfix 分支**: Bug 修复分支，命名: `bugfix/问题描述-开发者`
- **hotfix 分支**: 紧急修复分支，命名: `hotfix/问题描述-开发者`

### 11.2 提交规范

**提交信息格式**:
```
<类型>(<范围>): <简短描述>

[可选的详细描述]

[可选的关联 Issue]
```

**类型分类**:
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具更新

```
feat(dataflows): 添加港股数据获取模块

实现港股市场数据获取功能，支持涡轮、牛熊证等衍生品数据。
支持 00700.HK、09988.HK 等主要股票。
```

## 十二、项目开发的强制要求

### 12.1 开发环境要求

- **Python 版本**: >= 3.13
- **虚拟环境**: 必须使用 virtualenv/conda/poetry 管理依赖
- **IDE**: 推荐使用 VS Code 或 PyCharm
- **必需插件** (VS Code):
  - Python
  - Pylance
  - Black Formatter
  - isort
  - mypy

### 12.2 代码质量强制要求

- **必须通过**:
  - Black 格式化检查
  - isort 导入排序检查
  - mypy 类型检查 (严格模式)
  - pytest 测试 (通过率 100%)
- **覆盖率要求**:
  - 新增代码覆盖率 >= 80%
  - 核心模块覆盖率 >= 90%
- **审查要求**:
  - 必须有代码审查
  - 审查者必须实际运行代码验证

### 12.3 安全要求

- **敏感信息**:
  - 禁止提交 API 密钥、密码等敏感信息到仓库
  - 使用 .env 文件管理环境变量 (已配置 .gitignore)
  - 敏感信息使用环境变量或 GitHub Secrets
- **依赖安全**:
  - 定期运行 `pip audit` 检查漏洞
  - 使用 Dependabot 自动更新依赖
- **数据安全**:
  - 用户数据必须加密存储
  - 日志中禁止记录敏感信息

### 12.4 测试要求

- **单元测试**:
  - 所有工具函数必须有单元测试
  - 边缘情况必须覆盖
- **集成测试**:
  - API 接口必须集成测试
  - 数据流必须端到端测试
- **回归测试**:
  - Bug 修复必须添加回归测试
  - 重大变更必须回归测试

### 12.5 文档要求

- **代码内文档**:
  - 所有公共 API 必须包含 docstring
  - 复杂逻辑必须有行内注释
- **项目文档**:
  - 新功能必须在 README 中更新
  - API 变更必须更新文档
- **变更记录**:
  - 每个版本必须更新 CHANGELOG
  - 重大变更必须记录迁移方案

### 12.6 性能要求

- **响应时间**:
  - 数据获取 API 响应时间 < 5 秒
  - Agent 推理响应时间 < 30 秒
- **资源限制**:
  - 单次请求内存使用 < 500MB
  - 避免无限循环，添加超时机制
- **缓存策略**:
  - 热门数据必须缓存
  - 缓存过期时间合理设置

### 12.7 合规要求

- **法律声明**:
  - 项目必须包含免责声明
  - 明确说明不提供投资建议
- **许可证**:
  - 使用 MIT/Apache 2.0 许可证
  - 第三方库许可证兼容
